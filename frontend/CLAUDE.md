This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

React 19 + TypeScript 5.9 + Vite 7 frontend for a Telegram Mini App that does AI-powered food photo analysis and calorie tracking. Uses Bun as the package manager and runtime. Deployed on Vercel. The backend is a separate FastAPI service in `../backend/`.

## Commands

```bash
bun run dev                # Vite dev server on port 3000
bun run build              # tsc -b && vite build
bun run check              # type-check + lint + format:check (run before committing)
bun run lint               # ESLint (max 10 warnings allowed)
bun run format             # Prettier
bun run api:gen            # Generate API client from OpenAPI spec via Orval
bun run i18n:validate      # Verify EN/RU translation key parity

# Testing
bun run test                                          # All unit tests (vitest)
bun run vitest run src/path/to/file.test.ts           # Single test file
bun run vitest run src/path/to/file.test.ts -t "name" # Single test
```

## Architecture

### Entry & Routing

`src/main.tsx` initializes the Telegram WebApp SDK (`expand()`, `ready()`, sets header/bg colors), imports i18n, then mounts the React app.

`src/app.tsx` is the core component: sets up React Router with 4 lazy-loaded pages (`/` -> Meals, `/stats`, `/goals`, `/feedback`), wraps everything in `TelegramProvider`, `QueryClientProvider`, `ErrorBoundary`, and `AppLayout`.

### Telegram Integration

`src/providers/telegram.tsx` — `TelegramProvider` handles all Telegram environment sensing in one place:
- Theme detection (Telegram SDK, system `prefers-color-scheme`, manual override)
- Language detection (Telegram user data, browser locale, stored preference)
- Safe area handling (device notches/home indicators)
- User context (user ID, theme, language, safe areas)

User ID is extracted from `window.Telegram?.WebApp?.initDataUnsafe?.user?.id` with fallbacks.

### API Layer

Generated by **Orval** from the backend OpenAPI spec. Configuration in `orval.config.ts`.

- `src/api/model/` — TypeScript types generated from OpenAPI schemas
- `src/api/queries/` — TanStack Query hooks generated from OpenAPI endpoints
- `src/api/client.ts` — Custom fetch client that attaches `X-Correlation-ID`, `x-user-id`, and `Authorization` headers

Run `bun run api:gen` to regenerate after backend API changes. CI verifies generated code is up-to-date.

### Server State Management

**TanStack Query** (`@tanstack/react-query`) manages all server state:
- Generated query hooks in `src/api/queries/` for data fetching
- Automatic caching, background refetching, and error handling
- No manual `useState`/`useEffect` for API data

### Styling

**Tailwind CSS v4** with `@theme` directive for design tokens. All colors reference Telegram CSS variables (`--tg-*`), never hardcoded. Configuration in `src/app.css` using the CSS-first approach (no `tailwind.config.js`).

### Config

`src/config.ts` — centralized config from `import.meta.env`. Access via `config.apiBaseUrl`, `config.features.*`, etc.

### i18n

`src/i18n/index.ts` — i18next with nested keys. Translations split into `src/i18n/en.ts` and `src/i18n/ru.ts`. All user-facing text must use `t('key')` from `useTranslation()`. Run `bun run i18n:validate` after adding/changing keys.

### Testing

- Test setup: `src/test/setup.ts` — mocks localStorage, matchMedia, ResizeObserver, IntersectionObserver, Telegram WebApp
- Test utilities: `src/test/utils.tsx` — `renderWithProviders()` wraps components in QueryClient, Router, i18n
- MSW handlers: `src/test/handlers.ts` — mock API responses
- Framework: Vitest + React Testing Library

## Key Conventions

- **Path alias:** `@/*` maps to `src/*` in imports
- **Unused variables:** prefix with `_` to suppress ESLint warnings
- **Lazy loading:** all pages use `React.lazy()` + `Suspense`
- **CSS theming:** all colors reference Telegram CSS variables (`--tg-*`), never hardcoded
- **API types:** always use Orval-generated types from `src/api/model/`, never define API types manually
- **Server state:** use TanStack Query hooks from `src/api/queries/`, no manual fetch + useState
- **Package manager:** Bun (not npm/yarn). Use `bun install`, `bun run`, `bun add`
